# Pitfalls — battle scars

Bugs we've already paid for. Open this file before:
- Adding retries / timeouts to network calls
- Touching cross-frame messaging
- Adding listeners in content scripts
- Writing DB transactions
- Returning error responses

Migrated from `docs/HANDOFF.md` §6. Future pitfalls land here via `/learn`.

---

- [2026-05-12] (audio) Whisper segment timestamps drift ±1s within a 10s chunk. We accept this; do NOT try to reconcile across chunks. Reason: chunk-uniform was worse. last-cited: 2026-05-12
- [2026-05-12] (audio) Comparing wall-clock chunk advancement to `video.currentTime` via sample-rate math BREAKS at 2× playback (resets every chunk → infinite loop). Use `lastObservedVideoTime` jump detection (>2s) instead. Reason: playback-rate-agnostic = only safe form. last-cited: 2026-05-12
- [2026-05-12] (cross-frame) K-LMS / Vimeo / Canvas Studio embed in cross-origin iframes. Modal mounts in TOP, capture in IFRAME. Coordination via `window.postMessage` with `source: 'sh-frame'` (iframe→top) and `source: 'sh-parent'` (top→iframe). Top relays modal control msgs to iframes. Reason: silent no-op if source tag missing. last-cited: 2026-05-12
- [2026-05-12] (content-script) SPA navigations re-run content scripts in the same document. `__SH_CONTENT_BOOTED__` window sentinel guards listener registration. Without it, MutationObservers + chrome.runtime listeners stack on every nav. Reason: zombie listeners. last-cited: 2026-05-12
- [2026-05-12] (api-gw) API Gateway HTTP 30s timeout is HARD — cannot raise. Curator uses Lambda Function URL to bypass. Do NOT put long-running handlers behind API GW. Reason: physics. last-cited: 2026-05-12
- [2026-05-12] (db) DB Pool is `max:2`. Multi-statement transactions need `pool.connect()` + same-client query for BEGIN/COMMIT (see `lib/migrate.ts`). `pool.query` calls get DIFFERENT connections → BEGIN/COMMIT on different conns → transaction is a no-op. Reason: data loss. last-cited: 2026-05-12
- [2026-05-12] (zod) `withAuth` wrapper catches `ZodError` → 400. New handlers should call `Body.parse()` inside the inner function (it'll throw; wrapper handles). Don't try/catch around `.parse()` in the handler. Reason: divergence in error format. last-cited: 2026-05-12
- [2026-05-12] (api) Slide upload (`stream-slide`) body MUST include `url`. Same Zod schema as `stream-audio`. Easy to forget when adding new endpoints — they share the shape. Reason: was a 500 for a week. last-cited: 2026-05-12
- [2026-05-12] (api) API JSON responses MUST set `Content-Type: application/json`, especially 4xx/5xx. Frontend SW JSON-parses every response; without the header browser may treat as text/plain. Reason: inconsistent parse failures. last-cited: 2026-05-12
- [2026-05-12] (cors) Function URL CORS is a SEPARATE config from API GW CORS. Both must be locked post-publish. Both currently `*` pre-launch. Reason: forgetting one = data leak. last-cited: 2026-05-12
- [2026-05-12] (network) `stream-audio` chunk POST retries once after 1.5s on 5xx. Mitigates the ~22% API-GW-upstream-503 rate observed in real usage. Don't increase retry count without re-measuring — duplicate STT cost. Reason: budget + idempotency. last-cited: 2026-05-12
- [2026-05-24] (css-shadow) `filter: drop-shadow(x y blur)` bleeds upward by `max(blur - y, 0)`. ALWAYS keep `y ≥ blur` for cast shadows, otherwise the shadow extends above the element edge and looks like a halo. Affected: `.postit` shadow stack — V2-B uses `y = blur` (0.3em/0.3em, 0.7em/0.7em, 1.4em/1.4em). Reason: confirmed via visual verification loop on PR #32. last-cited: 2026-05-24
- [2026-05-24] (css-stacking) Drawing a vertical / horizontal page-margin line with `::before { position: absolute; z-index: -1 }` on a `position: relative` parent does NOT keep it below child sections — `transform` / `isolation` create a stacking context that traps `z-index: -1` above the parent background but BELOW nothing else, so children with their own `background-color` still cover it inconsistently. Use `background-image: linear-gradient(...)` on the parent instead (a solid 1.5px strip) so any child with a background-color naturally covers it. Reason: the legal-pad red margin should only show on transparent paper sections, not on PrivacyEmphasis (dark) / pricing cards / footer. last-cited: 2026-05-24
- [2026-05-24] (css-responsive) `box-shadow` / `filter: drop-shadow` values in fixed `px` look wrong when an element scales across viewports — at small sizes the shadow is too heavy, at large sizes it's too small. Express all shadow values in `em` and set the element's `font-size: clamp(min, vw, max)`. Then text content uses explicit px so font-size only drives shadow scale. Example in `.postit`. Reason: visual proportionality across mobile / tablet / desktop. last-cited: 2026-05-24
- [2026-05-26] (vitest-discovery) Vitest 4 dropped `**/dist/**` from `configDefaults.exclude` (present through v3). When a TS package's tsconfig includes `tests/**` AND `outDir` is `dist`, `pnpm build` emits `dist/tests/**/*.test.js` duplicates and vitest runs both copies; the dist copies resolve `__dirname`-relative FS reads into `dist/<asset>` where `tsc` never copies JSON/SQL → ENOENT. Add per-package `vitest.config.ts` with `exclude: [...configDefaults.exclude, 'dist/**']` (pattern already used by `web/`, `desktop/`). Reason: latent since the vitest 4 bump; surfaced 2026-05-26 by PR #45 Plan A — A5/A6 were the first code paths to read assets via `__dirname`, ~30 min CI block. last-cited: 2026-05-26
- [2026-05-27] (spike-llm) Spike-spawned LLM processes (`llama-completion`, vitest workers) MUST be terminated before session end: use `afterAll` / `try/finally` in tests, verify with `ps -ef | grep -E "llama-completion|vitest.*spike" | grep -v grep` and `kill -9` survivors. Never use `run_in_background:true` for heavy LLM inference — session resume auto-restarts it, stacking processes. Reason: 8GB M3 sustained Llama-3.2-3B → 3GB RAM zombie → swap thrashing on next session start → kernel panic. last-cited: 2026-05-27 (re-cited: Plan 3 Task 16 dir-glob+background incident)
- [2026-05-27] (test-headers) When trimming test scope (sample count, fixture count, RAM budget, parallelism), update the file-header docstring AND any referenced spec/plan acceptance criteria IN THE SAME COMMIT as the code edit. Stale header asserts a wrong gate to the next reader — spike-0.1 `round-trip.test.ts` lines 1-7 stayed at "10/10 → STOP" after the N=10→5 trim in commit `251c1fc`, propagating through `46ed08a` "HARD GATE cleared" with the header still contradicting the assertion, requiring follow-up commits `9eda9b1` + `060d1fd` to align Plan + spec + memo + header. Reason: passing-run interpretation depends on the header. last-cited: 2026-05-27
- [2026-05-27] (vitest-scope) `pnpm vitest run` with no filter OR with a spike directory filter (e.g. `spikes/phase-0/01-zod-to-gbnf/`) both pick up `round-trip.test.ts` (hardware-gated real-LLM, ~16 min) — the dir mixes it with pure unit tests. For verification gates pass explicit FILE paths only (`spikes/.../zod-to-gbnf.test.ts`) or restrict to `src/`. Reason: Plan 2 Task 19 burnt 16 min (root filter); Plan 3 Task 16 lost ~2 min + 3 LLM iterations on 8GB M3 (dir filter + background). last-cited: 2026-05-27
- [2026-05-28] (pre-push-lint) Pre-push verification AND every SDD implementer/reviewer dispatch prompt MUST run `pnpm lint` (or full `pnpm verify`), not just `tsc --noEmit` + scoped vitest — eslint catches unused vars/imports tsc ignores, and `desktop-ci` gates on `pnpm verify` (which runs lint). Reason: recurred twice — Plan 4 (#59) + Plan 5 (#60), each shipped an unused import/param that passed tsc + tests but failed eslint in CI. last-cited: 2026-05-28
- [2026-05-28] (llm-overflow) On-device LLM generate() SILENTLY truncates/empties on n_ctx overflow — sidecar breaks its decode loop (llama_engine.cpp:201 `if (llama_decode(...) != 0) break;`) but still emits a normal `done`, so TS sees a clean but empty/truncated result with NO error. Never feed an unbounded transcript into one generate() pass; chunk to stay within n_ctx (live session/stop fixed in chunked-note.ts, PR #66). Reason: ~20min+ dense JA recordings silently lost half/all of the note. last-cited: 2026-05-28
- [2026-05-29] (node-stream) Reading a child-process stdout/stderr stream by concatenating `chunk.toString()` per `data` event corrupts multi-byte UTF-8 at chunk boundaries → U+FFFD. Collect raw Buffers and decode ONCE via `Buffer.concat(chunks).toString('utf8')` (or a `StringDecoder`). Reason: a 3-byte JA char split across two stdout chunks became `��` (8× in one Plan 6 Phase B spike chunk-0, `llama-cli-rig.ts`), misdiagnosed as a model/grammar failure and chased for a full session. last-cited: 2026-05-29
- [2026-05-29] (llm-grammar) A grammar-constrained INSTRUCT model fed a RAW prompt (no chat template) goes out-of-distribution: it won't emit EOS so it runs on until `maxTokens` truncates the JSON mid-string. The resulting `JSON.parse` "Bad control character in string literal at position N" is the TRUNCATION symptom (llama-completion's `> EOF by user` epilogue newline inside the unterminated string), NOT a grammar char-class gap. Apply the model's chat template (`--jinja`/embedded, mirrors the production sidecar) + strip the epilogue before parse. Reason: a prior session misread this as a GBNF char-rule bug (F1) and patched the grammar, which could not fix it (Plan 6 Phase B). last-cited: 2026-05-29
- [2026-05-30] (react-reserved-props) React strips `ref` from function-component props (it's reserved for forwardRef); custom components that need a numeric speaker-reference / row-id / similar MUST use a different prop name (`speakerRef`, `rowId`, …) — `ref={n.owner}` silently renders nothing because the function never sees the value. Reason: caught in TDD on MeetingRenderer's `SpeakerTag` (#76); the same shape will recur in Plan 6 Interview/Brainstorm renderers (qa_pairs[].from, ideas[].proposed_by, …). last-cited: 2026-05-30
- [2026-06-11] (gha-cache) A GitHub Actions job-timeout kill counts as "cancelled" and SKIPS `actions/cache@v4` post-job saves — so a cache whose fill step can hang NEVER seeds, and every run/rerun re-downloads and hangs again (self-perpetuating deadlock on a required check). Split into `actions/cache/restore` + an explicit `actions/cache/save` placed immediately after the fill step, and give the fill step its own step-level `timeout-minutes`. Reason: required `ci` check was red for 3 days (2026-06-08~11) — Playwright <1.60.0 install hang on Node 24.16+ (microsoft/playwright#40998); fixed in PR #109. last-cited: 2026-06-11
