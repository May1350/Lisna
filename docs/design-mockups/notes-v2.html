But what is a neural network?

updated 2m ago
TL;DR Neural Network 은 인간 뇌의 시각 인식 능력을 모방하기 위한 계층적 연산 구조. 28×28 pixel 의 손글씨 숫자 인식 task 를 통해 input → hidden layers → output 의 정보 흐름과, 각 neuron 이 weighted sum + activation 으로 동작하는 원리를 설명한다. 학습이란 weights 와 biases 를 조정해 올바른 인식 결과를 얻는 과정이다.

01

脳の視覚認識能力という驚異

脳は異なる解像度・スタイルの画像を同じ物体として認識できる。28×28 pixel の極小画像でも余裕で「3」と判別する能力こそ、neural network が模倣しようとしている対象である。

Take 低解像度で大きく異なる見た目でも同じ概念として認識できる「汎化」が、人間の視覚認識の本質
  • 28×28 pixel = 784 個の輝度値だけで「3」と判別可能
  • スタイル・大きさ・解像度に依存しない identification
  • この汎化能力こそ neural network が模倣を目指す

02

Neural Network の基本構造

NN は input layer → hidden layers → output layer の階層構造をもつ。各 layer の neuron は前 layer からの値を重み付き和として受け取り、activation function を通して次に伝える。

Take Neuron は単なる「0〜1 の値を持つ箱」ではなく、前 layer 全体からの加重和を計算する関数
Neuron 0〜1 の値 (activation) を保持する計算単位。input layer では pixel の輝度値そのまま。
Layer 同じ階層に並ぶ neuron の集合。input → hidden → output の順に flow する。
Weighted Sum 前 layer の各 activation に対応する weight を掛け、すべて足し合わせた値。
  • Input layer: 784 個 (28×28 pixels)
  • Hidden layers: 16 + 16 (任意の選択)
  • Output layer: 10 個 (digit 0〜9)
architecture layer neuron
03

数字認識タスクの定義

MNIST dataset を例に、28×28 grayscale 画像 1 枚を input として受け取り、最も「らしい」digit (0〜9) を output する分類問題として定式化する。

  • Input: 784 次元のベクトル (各成分 ∈ [0, 1])
  • Output: 10 個の neuron の中で最大値をもつものを「答え」とする
  • 学習データ: ラベル付き 60,000 枚の手書き数字
mnist classification
04

なぜ Layer が必要か — Pattern の階層

複雑な認識タスクは、より単純なサブタスクの組み合わせとして分解できる。Hidden layer はこの分解を表現する場所であり、後段に行くほど抽象的な特徴 (edge → loop → digit) に対応すると期待される。

Take Layer の存在意義 = 「単純 pattern → 中間 pattern → 最終判定」 の階層的構造を学習で表現するため
Edge 画像中の輝度の急変化。最も低レベルな特徴。
Loop edge を組み合わせた閉曲線。"8" は 2 つの loop、"9" は loop + 縦線。
Hierarchy Edge → Loop → Digit のように、抽象度が段階的に上がる pattern の階層。
  • 数字 "9" = 上の loop + 下の縦線 という構造分解
  • "8" = 上の loop + 下の loop
  • "4" = 縦線 2 本 + 横線 1 本
  • この分解こそ hidden layer が学ぶべき内容
hierarchy edge detection composition
Check "6" を edge と loop の組み合わせで分解せよ。
05

Edge Detection の仕組み

Hidden layer の neuron は、「特定の領域に edge が存在するか」を検出するように weight を設定できる。検出したい edge 部分の weight を正、それ以外を負にすれば、weighted sum はその edge の有無に応じた値となる。

Take Weight を「正 / 負 / 0」に配置することで、neuron は特定の edge pattern detector として機能できる
  • 特定 pixel 群に正の weight → そこが明るいと neuron 値が上がる
  • 周囲に負の weight → 「edge であって全面的ではない」状態を要求
  • これを Sigmoid に通せば 0〜1 の「edge 存在確率」になる
weights edge detector
06

Bias と Sigmoid Function

Weighted sum をそのまま活性化に使うと範囲が広すぎる。Sigmoid で [0, 1] に圧縮し、bias を加えることで「ある閾値を超えたら活性化する」仕組みを表現する。

Sigmoid σ(x) = 1 / (1 + e⁻ˣ). 任意の実数を [0, 1] に圧縮する S 字曲線。
Bias 活性化の閾値を調整するスカラー。weighted sum に加算してから sigmoid を通す。
  • activation = σ(Σ wᵢxᵢ + b)
  • Bias が大きく負 → 高い値が必要 → 検出に厳しい neuron
  • Bias が大きく正 → 緩い neuron
  • 現代では sigmoid より ReLU が主流 (本講義は理解のため sigmoid)
sigmoid bias activation function
07

パラメータ数 — 学習対象の規模

Layer 構成 784 → 16 → 16 → 10 では、weights が 784×16 + 16×16 + 16×10 = 13,002 個、biases が 16 + 16 + 10 = 42 個。合計 13,044 個のパラメータを学習で調整する必要がある。

Take 「学習する」とは、この 13,044 個のダイヤルを正しい組み合わせに合わせる作業に他ならない
  • Weights: 13,002 (= 784×16 + 16×16 + 16×10)
  • Biases: 42 (= 16 + 16 + 10)
  • 合計 13,044 個 — 全部を手動で設定するのは不可能
  • Backpropagation により自動で調整 (次回テーマ)
parameters scale
Check Layer 構成 784 → 32 → 10 のときの total parameter 数を計算せよ。
08

行列表現とまとめ

Layer 間の伝搬は a' = σ(Wa + b) という行列演算でコンパクトに書ける。これは GPU での並列計算と相性が良く、現代の deep learning の高速化基盤となっている。

  • a' = σ(Wa + b) — 全ての neuron の更新を一行で表現
  • W は前 layer × 次 layer サイズの行列
  • この簡潔さこそ NN が GPU で高速化される理由
matrix linear algebra gpu
Check a' = σ(Wa + b) の各記号 (a, W, b, σ) の役割を 1 行ずつ説明せよ。
