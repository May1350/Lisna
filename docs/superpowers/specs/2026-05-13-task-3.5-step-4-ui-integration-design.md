## 1. Overview + scope

### 1.1 Problem

`SessionOrchestrator` (Steps 1-3, commits `d371f49 / 365413b / d3e9a69 / ceaff0f`) is implemented and tested in isolation, but **nothing in the running app calls it**. The current IPC chunk path goes directly through `deps.stt.transcribe()` (`desktop/src/main/ipc.ts:40`), bypassing the orchestrator entirely. The `recording/stop` IPC is a `{ ok: true }` stub, and there is no UI surface for the resulting Note.

Step 4 wires the orchestrator end-to-end:
1. Renderer's stop button triggers a real `session/stop` IPC that returns a `Note`.
2. Renderer routes the Note to a new `NoteView` component.
3. Recording chunks flow through the orchestrator's `onChunk` so segments are owned in one place.
4. STT loading moves from app boot to session start (8GB RAM constraint requires STT and LLM not coexist resident).

### 1.2 What the user sees after Step 4

- Click **Start** → "Loading model..." → live captions appear → click **Stop** → "Generating note..." (phase indicator) → NoteView with markdown + transcript collapsible.
- On error (sidecar crash, LLM OOM, generate timeout): ErrorView with message + transcript fallback + Try again button.

### 1.3 Out of scope

See §9 (Deferred + risks). High-level cuts: react-markdown for NoteView, real-time token streaming, multi-language settings UI, partial-Note persistence on quit, mic-permission gate hardening.

## 2. Architectural fork — chunk routing through orchestrator

The single biggest decision: **the `recording/chunk` IPC handler now routes through `SessionOrchestrator.onChunk()`** rather than calling `deps.stt.transcribe()` directly. Consequences:

- The orchestrator becomes the single owner of `segments[]` (already the case inside `orchestrator.ts:40`).
- The orchestrator must exist before chunks arrive → an explicit `session/start` IPC creates it.
- App-boot eager STT load (`main/index.ts:43-58`) is removed; STT loads at `session/start` time only.
- The orchestrator's single-use FSM contract (JSDoc-enforced in `orchestrator.ts:18-38`) is enforced by the IPC handler layer via synchronous `current` / `recording` flags.

## 4. Main process FSM + crash recovery

### 4.1 State

```ts
// desktop/src/main/ipc.ts
let current: SessionOrchestrator | null = null;
let recording = false;
let _appQuitting = false;

export interface IpcDeps {
  getMainWindow: () => BrowserWindow | undefined;
  supervisor: SidecarSupervisor;
  sttModelPath?: string;
  llmModelPath?: string;
}
```

`current` / `recording` are mutated synchronously around `await` points. **No mutex** — JS single-threaded execution between awaits is the synchronization primitive. The sidecar's NDJSON FIFO is the other layer of serialization (it already orders sidecar requests).

### 4.2 State transitions

```
idle  (current=null, recording=false)
  │
  │ session/start (success)
  ▼
ready  (current=orch, recording=true)
  │
  │ session/stop entry (sync: recording=false)
  ▼
finalizing  (current=orch, recording=false, stop awaiting)
  │
  │ orch.stop resolves / throws (finally: current=null)
  ▼
idle

Sub-states (transient, between syncs and awaits):
  starting  = current=orch, recording=false, session/start awaiting orch.start()

Crash edge (any state with current !== null):
  handleSidecarExit() → current=null, recording=false, push session/error → idle
```

### 4.3 Handler bodies

All four handlers are registered inside `registerIpc(deps: IpcDeps)`. They and the `safeSend` helper close over `deps` for `getMainWindow` / `supervisor` / `sttModelPath` / `llmModelPath` access. Module-level state (`current`, `recording`, `_appQuitting`) is mutated directly.

```ts
// inside: export function registerIpc(deps: IpcDeps) { ... }
ipcMain.handle('session/start', async (_e, { language }: SessionStartPayload) => {
  if (current !== null) throw new Error('SESSION_ACTIVE');
  if (language !== 'ja') throw new Error('UNSUPPORTED_LANGUAGE');  // v2.0 JA-only
  if (!deps.sttModelPath || !deps.llmModelPath) throw new Error('MODELS_NOT_CONFIGURED');
  const client = deps.supervisor.getClient();
  if (!client) throw new Error('SIDECAR_DOWN');
  // Fresh adapters per session — survives sidecar respawn (no stale client refs).
  // WhisperCppSTT / LlamaCppLLM constructors are pure: just stash this.client,
  // no listener registration, no internal state. Per-session construction is cheap.
  const stt = new WhisperCppSTT(client);
  const llm = new LlamaCppLLM(client);
  const orch = new SessionOrchestrator({
    stt, llm,
    sttModelPath: deps.sttModelPath,
    llmModelPath: deps.llmModelPath,
    language,
  });
  current = orch;  // claim BEFORE await — re-entry blocked synchronously
  try {
    safeSend('session/phase', { phase: 'stt-loading' });
    await orch.start();
    recording = true;
  } catch (err) {
    console.error('[session] start failed', err);
    current = null;
    throw err;
  }
});

ipcMain.handle('recording/chunk', async (e, payload: ChunkPayload) => {
  if (!recording || !current) return { ok: true };  // start in flight / post-stop: silent no-op
  const orch = current;
  try {
    const segs = await orch.onChunk(payload.samples);
    e.sender.send('recording/chunk-result', {
      index: payload.index,
      segments: segs,
      startMs: payload.startMs,
    } satisfies ChunkResultPayload);
  } catch (err) {
    // One failed chunk must not break the session — same policy as current handleChunk.
    console.error('[stt] chunk transcribe error', payload.index, err);
  }
  return { ok: true };
});

ipcMain.handle('session/stop', async (_e) => {
  if (current === null) throw new Error('NO_ACTIVE_SESSION');
  if (!recording) throw new Error('SESSION_NOT_READY');  // start in flight
  const orch = current;
  recording = false;  // sync: post-stop chunks immediately no-op
  try {
    return await orch.stop((phase) => safeSend('session/phase', { phase }));
  } catch (err) {
    if (_appQuitting) throw new Error('APP_QUIT');
    throw err;
  } finally {
    current = null;
  }
});

  // helpers defined inside registerIpc to close over `deps`:
  function safeSend(channel: string, payload: unknown) {
    const w = deps.getMainWindow();
    if (!w || w.isDestroyed()) return;
    w.webContents.send(channel, payload);
  }
} // end registerIpc

// Module-level exports (mutate the module-private flags above):
export function setAppQuitting() { _appQuitting = true; }

export function handleSidecarExit() {
  if (current || recording) {
    current = null;
    recording = false;
    // safeSend is closure-bound to deps inside registerIpc — to call it from
    // module-level, ipc.ts holds a `_safeSend` ref captured during registerIpc.
    // (Implementation detail: store `_safeSend` at module scope, assign inside
    // registerIpc; handleSidecarExit calls _safeSend.)
    _safeSend?.('session/error', {
      message: 'Recording engine restarted. Please try again.',
    });
  }
}

let _safeSend: ((channel: string, payload: unknown) => void) | null = null;
// (Inside registerIpc, after defining safeSend: `_safeSend = safeSend;`)
```

### 4.4 Supervisor crash recovery

The `SidecarSupervisor` auto-respawns the sidecar on unexpected exit (up to `maxConsecutiveFailures`, default 2). The existing `onCrash` callback fires only after give-up. Step 4 adds a per-exit hook.

**Non-breaking addition to `SupervisorOptions`** (`desktop/src/main/sidecar/supervisor.ts`):

```ts
interface SupervisorOptions {
  onCrash: (msg: string) => void;     // existing — give-up signal
  onExit?: () => void;                // NEW — every unexpected exit
  // existing fields unchanged
}
```

`handleExit` body change (insert after `this.proc = undefined`, before the `shuttingDown` early return):

```ts
private handleExit(code: number | null, sig: NodeJS.Signals | null): void {
  if (this.healthyResetTimer) { clearTimeout(this.healthyResetTimer); ... }
  this.client = undefined;
  this.proc = undefined;
  if (this.shuttingDown) return;
  this.onExit?.();                                // ← NEW: before failure-counter logic
  this.failuresInARow += 1;
  if (this.failuresInARow >= this.maxFailures) {
    this.onCrash(`Sidecar exited ${this.failuresInARow} times in a row ...`);
    return;
  }
  setTimeout(() => { if (!this.shuttingDown) this.start(); }, this.restartDelayMs);
}
```

**Listener registration order invariant** (`supervisor.start()`, lines 67-69): `new SidecarClient(this.proc)` MUST register before `this.proc.on('exit', handleExit)`. Node fires listeners in registration order, so `client.rejectAllPending` runs first, which synchronously rejects the in-flight `orch.stop()` / `orch.onChunk()` work — that lets the `finally { current = null }` in `session/stop` complete before `handleSidecarExit` sees state. Without this order, `handleSidecarExit` would clobber state mid-`finally`, double-pushing `session/error` while the renderer already received a handler rejection. **Add a comment in `supervisor.ts:67-69`** pinning this order: *"Do not reorder: client must register its exit handler before supervisor's so pending rejects flow before handleExit fires."*

**Two paths into `session/error` reduced to one source:**

- `onCrash` callback (give-up) — **log-only** (`console.error('[sidecar give-up]', msg)`). No renderer push from this path.
- `onExit` callback (every exit, including give-up cases) — **source of truth for renderer notification** via `handleSidecarExit`.

This avoids the double-push that would occur if both callbacks emitted `session/error` (renderer would flip to error view twice with conflicting copy on give-up cases, since `onExit` fires before `onCrash` in `handleExit`).

### 4.5 Race walkthrough

| Scenario | Behavior |
|---|---|
| Chunk arrives after `session/stop` dispatched | Chunk sees `recording === false` (sync set at stop entry — before stop handler awaits) → silent no-op. |
| `session/start` while `session/stop` in flight | Second start sees `current === orch` → `SESSION_ACTIVE` reject. |
| `session/stop` invoked twice rapidly | Second sees `current === null` after first's `finally` → `NO_ACTIVE_SESSION` reject. |
| Chunk in-flight when sidecar crashes | client's `proc.on('exit')` rejects pending → chunk handler's catch swallows + logs → renderer's segment list unchanged for that chunk. `handleSidecarExit` clears flags + pushes `session/error`. |
| `session/stop` in-flight when sidecar crashes | client's listener (registered FIRST) rejects pending → `orch.stop` throws → `session/stop` IPC promise rejects → renderer's `.catch` triggers error view. `finally { current = null }` runs SYNC. THEN supervisor's listener fires `onExit` → `handleSidecarExit` sees `current === null` → skip push. Single error transition. |
| Chunk in-flight when `session/stop` invoked | `recording=false` sync. In-flight chunk's `onChunk` completes transcribe (sidecar FIFO serializes against unloadModel). Stop button feels up-to-5s laggy worst case. Accepted v2.0 latency. |
| User clicks Start during 500ms respawn delay | `supervisor.getClient()` returns undefined → `SIDECAR_DOWN` reject → renderer shows ErrorView. User clicks Try Again — likely works on second attempt. |
| `app.quit` mid-finalize | `setAppQuitting()` fires synchronously in before-quit. `supervisor.shutdown()` SIGTERMs sidecar. `orch.stop` rejects (sidecar dead). `session/stop` catch remaps to `APP_QUIT`. Renderer ignores `APP_QUIT` (window dying anyway). |

### 4.6 In-flight chunk + unloadModel overlap (known v2.0 latency)

When `session/stop` enters, a chunk's `onChunk` may have already captured `current` and be inside `stt.transcribe()`. The orchestrator's `stop()` will then call `stt.unloadModel()` — two STT requests overlap in the sidecar. The sidecar's NDJSON FIFO serializes at the transport layer, but whisper.cpp's internal serialization is unverified. v2.0 accepts the worst case: Stop button awaits the in-flight transcribe (5-10s on M1 8GB) before proceeding to `unloadModel`. Added sidecar-side serialization test is deferred to Task 3.6+ (see §9).

## 5. SessionOrchestrator API extension

### 5.1 Signature change (`desktop/src/main/sidecar/orchestrator.ts`)

Non-breaking add: optional `onPhase` callback parameter to `stop()`.

```ts
async stop(onPhase?: (phase: SessionPhase) => void): Promise<Note> {
  try {
    onPhase?.('stt-unloading');
    await this.opts.stt.unloadModel();
    onPhase?.('llm-loading');
    await this.opts.llm.loadModel(this.opts.llmModelPath);
    onPhase?.('generating');
    const prompt = (this.opts.buildPrompt ?? defaultPrompt)(this.opts.language, this.segments);
    let md = '';
    for await (const tok of this.opts.llm.generate(prompt, { maxTokens: 4096, temperature: 0.4 })) md += tok;
    return {
      language: this.opts.language,
      generatedAt: new Date().toISOString(),
      markdown: md,
      transcriptSegments: this.segments,
    };
  } finally {
    await this.opts.llm.unloadModel().catch(() => {});
  }
}
```

### 5.2 Semantics

- Callbacks fire **before** each async phase — they mark "starting this phase" not "completed". Renderer should show "Loading note-writing model..." while the load is happening.
- The final `llm.unloadModel()` in `finally` is intentionally silent. The renderer has already transitioned to NoteView (or ErrorView) by the time it runs.
- The caller emits `'stt-loading'` separately (around `start()` — done from `session/start` IPC handler).
- Callbacks are fire-and-forget. Return value ignored. Callback errors are not caught — caller must ensure non-throwing behavior.

### 5.3 JSDoc additions

Class-level JSDoc gets one new line at the end of the `stop()` description: *"Optional `onPhase` callback observes the three internal awaits (stt-unloading → llm-loading → generating) before each begins; the final `llm.unloadModel()` in `finally` is intentionally silent."*

`stop()` gets a `@param onPhase` block: *"Optional observer fired synchronously BEFORE each of the three internal awaits. Return value is ignored (fire-and-forget — do not await side effects inside). Callback errors are not caught; caller must ensure non-throwing. Caller emits 'stt-loading' around `start()` separately."*

## 6. Renderer flow

### 6.1 App.tsx state machine

```ts
// desktop/src/renderer/App.tsx
type View =
  | { kind: 'recording'; segments: TranscriptSegment[] }
  | { kind: 'finalizing'; phase: 'stt-unloading' | 'llm-loading' | 'generating'; segments: TranscriptSegment[] }
  | { kind: 'note'; note: Note }
  | { kind: 'error'; message: string; segments: TranscriptSegment[] };

export function App() {
  const [view, setView] = useState<View>({ kind: 'recording', segments: [] });

  // Chunk-result: accept in 'recording' AND 'finalizing'.
  // RecordingOrchestrator.stop()'s synchronous acc.flush() ships the final
  // partial chunk after the user clicks Stop. Its transcribe completes 50-500ms
  // later on main side — by then App is in 'finalizing'. Dropping that chunk-
  // result loses the last spoken sentence from both FinalizingView's transcript
  // and the final Note.transcriptSegments.
  useEffect(() => {
    return window.lisna.onChunk((msg) => {
      setView((prev) => {
        if (prev.kind === 'recording' || prev.kind === 'finalizing') {
          return { ...prev, segments: [...prev.segments, ...msg.segments] };
        }
        return prev;
      });
    });
  }, []);

  // Phase events: only act during 'finalizing'. The 'stt-loading' phase
  // emitted during session/start is ignored at App level — Recording's
  // local `starting` boolean drives the Start-button label.
  useEffect(() => {
    return window.lisna.onPhase(({ phase }) => {
      if (phase === 'stt-loading') return;
      setView((prev) => prev.kind === 'finalizing' ? { ...prev, phase } : prev);
    });
  }, []);

  // Session error (sidecar crash). Idempotent — if already in 'error',
  // preserve (first transition has the populated segments; a second call
  // from Recording.stop()'s catch must not overwrite with empty).
  useEffect(() => {
    return window.lisna.onSessionError(({ message }) => {
      setView((prev) => {
        if (prev.kind === 'error') return prev;
        const segments = (prev.kind === 'recording' || prev.kind === 'finalizing')
          ? prev.segments : [];
        return { kind: 'error', message, segments };
      });
    });
  }, []);

  switch (view.kind) {
    case 'recording':
      return <Recording
        segments={view.segments}
        onFinalizing={() => setView((prev) => prev.kind === 'recording'
          ? { kind: 'finalizing', phase: 'stt-unloading', segments: prev.segments }
          : prev)}
        onNote={(note) => setView({ kind: 'note', note })}
        onError={(message) => setView((prev) => {
          if (prev.kind === 'error') return prev;
          const segments = (prev.kind === 'recording' || prev.kind === 'finalizing')
            ? prev.segments : [];
          return { kind: 'error', message, segments };
        })}
      />;
    case 'finalizing':
      return <FinalizingView phase={view.phase} segments={view.segments} />;
    case 'note':
      return <NoteView note={view.note} onNewSession={() => setView({ kind: 'recording', segments: [] })} />;
    case 'error':
      return <ErrorView message={view.message} segments={view.segments} onRetry={() => setView({ kind: 'recording', segments: [] })} />;
  }
}
```

### 6.2 Recording.tsx changes

- `segments` prop (App owns state). Live captions `<ul>` renders from `props.segments`.
- `runningRef` removed — App-side chunk filter (`prev.kind === 'recording' || 'finalizing'`) is the new gate.
- `startingRef` kept — synchronous re-click guard within a single component instance (2026-05-13 smoke-test rationale).
- `starting` boolean kept — drives the Start-button "Loading model..." label.
- `chunks` counter removed (Phase 1 debug).
- Audio-first start ordering — `recording/start` → renderer-side `RecordingOrchestrator.start()` → `session/start` LAST. Rationale (inline comment in the code):

  2. If audio init fails after `session/start` succeeded, we'd need a `session/cancel` IPC to release STT — extra surface area we want to skip.
  3. Chunks during STT load are dropped server-side (`recording === false`). At 16kHz/2s chunks the lost window is ≤1 chunk; accepted v2.0.
- `language: 'ja'` hardcoded with `// TODO(v2.1): settings UI for language` comment.

## 7. Boot changes — `desktop/src/main/index.ts`

### 7.1 What's removed

Lines 43-58 of the current `main/index.ts`: eager STT load via `LISNA_DEV_STT_MODEL`, `WhisperCppSTT` boot construction, `adapter.loadModel`. The `let stt: STTEngine` assignment is gone. STT now loads only at `session/start` time.

### 7.2 Final shape

```ts
import { app, BrowserWindow } from 'electron';
import { fileURLToPath } from 'node:url';
import { dirname, join } from 'node:path';
import { registerIpc, handleSidecarExit, setAppQuitting } from './ipc';
import { installSystemAudioHandler } from './audio/system-audio-handler';
import { SidecarSupervisor } from './sidecar/supervisor';

const __dirname = dirname(fileURLToPath(import.meta.url));

let supervisor: SidecarSupervisor | undefined;
let mainWindow: BrowserWindow | undefined;
let shuttingDown = false;

function createWindow() {
  const win = new BrowserWindow({
    width: 1100, height: 720,
    webPreferences: {
      preload: join(__dirname, '../preload/index.mjs'),
      contextIsolation: true,
      sandbox: false,
    },
  });
  mainWindow = win;
  win.on('closed', () => { if (mainWindow === win) mainWindow = undefined; });
  if (process.env.ELECTRON_RENDERER_URL) win.loadURL(process.env.ELECTRON_RENDERER_URL);
  else win.loadFile(join(__dirname, '../renderer/index.html'));
}

app.whenReady().then(async () => {
  installSystemAudioHandler();

  supervisor = new SidecarSupervisor({
    onCrash: (msg) => console.error('[sidecar give-up]', msg),  // log-only breadcrumb
    onExit: handleSidecarExit,                                  // source of truth for renderer
  });
  const client = supervisor.start();
  client.onEvent((e) => console.log('[sidecar event]', e.type));

  try { await client.waitForReady(5000); }
  catch (err) { console.error('[sidecar] ready wait failed (recording will fail until restart):', err); }

  const sttModelPath = process.env.LISNA_DEV_STT_MODEL;
  const llmModelPath = process.env.LISNA_DEV_LLM_MODEL;

  createWindow();
  registerIpc({
    getMainWindow: () => mainWindow,   // getter — survives darwin re-create
    supervisor: supervisor!,
    sttModelPath,
    llmModelPath,
  });
});

app.on('before-quit', (event) => {
  if (shuttingDown) return;
  if (!supervisor) return;
  event.preventDefault();
  shuttingDown = true;
  setAppQuitting();
  supervisor.shutdown().finally(() => app.quit());
});

app.on('activate', () => {
  if (BrowserWindow.getAllWindows().length === 0) createWindow();
});

app.on('window-all-closed', () => app.quit());  // darwin included — v2.0 alpha policy
```

### 7.3 `mainWindow` lifecycle via getter

`registerIpc` receives a getter (`getMainWindow: () => BrowserWindow | undefined`) rather than a value. Every `safeSend` in `ipc.ts` reads the current ref. If the window is closed and recreated (darwin `activate` re-open), the next IPC push lands on the fresh window. Stale-ref bug avoided.

### 7.4 `window-all-closed` always quits

Default macOS behavior is "window close keeps app in dock." Step 4 overrides — `app.quit()` unconditional. Rationale: v2.0 alpha, single-window. Closing the window with a session active would leave `current` / `recording` flag stale in ipc.ts module scope. Quitting fully resets the process. v2.1 can re-introduce platform-conditional behavior once we have a menu-bar icon or background sync.

### 8.1 New: `desktop/src/main/sidecar/__tests__/supervisor.test.ts`

Tests against `registerIpc` with fake supervisor + fake orchestrator (use `vi.mock` for `WhisperCppSTT` / `LlamaCppLLM` adapters since they are constructed per-handler).

- `recording/chunk` before `session/start` → silent `{ ok: true }` no-op.
- `session/start` with missing env paths → rejects `MODELS_NOT_CONFIGURED`.
- `session/start` with `language: 'en'` → rejects `UNSUPPORTED_LANGUAGE`.
- `session/start` while `supervisor.getClient()` is undefined → rejects `SIDECAR_DOWN`.
- `session/stop` with `current === null` → rejects `NO_ACTIVE_SESSION`.
- Sidecar exit mid-recording (no in-flight chunk) → `handleSidecarExit` clears flags + pushes `session/error`.
- Sidecar exit mid-finalize → `session/stop` IPC rejects from in-flight reject path, `handleSidecarExit` sees `current === null` after `finally`, skips push. Renderer gets exactly one error transition.
- `setAppQuitting() + session/stop catch` → reject with `APP_QUIT`.

### 8.3 Extended: `desktop/src/main/sidecar/__tests__/orchestrator.test.ts`

- `fires onPhase callback in order during stop()` → events `['stt-unloading', 'llm-loading', 'generating']`.
- `still fires earlier phases when stop() throws mid-generate` → events still cover all three (mid-generate throw means 'generating' was emitted before).
- `emits only 'stt-unloading' when stt.unloadModel throws` → events `['stt-unloading']`, `fakeLlm.loadModel` NOT called. This is the behavior test that proves "before-the-await" semantics.

### 8.4 Renderer test gap

Recording.tsx / App.tsx / NoteView / ErrorView / FinalizingView — no unit tests for v2.0 MVP. Integration smoke (manual `pnpm dev` with real GGUF models) covers happy path. Post-alpha test pass adds component tests once the founder eval surfaces actual UX edges worth pinning.
