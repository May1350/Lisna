# TRACK 2 first STT scorecard — kotoba q5_0 (JA-30s, ±far-field-synth, 2026-06-09)

First STT WER/CER datapoint after wiring `realSttFn` into the
`runSttEval` orchestrator (PR #82). This is the founder-stated #1
user-visible quality gate per the TRACK 2 brainstorm. Pairs with the
LLM-side first scorecard in [PR #85](https://github.com/May1350/Lisna/pull/85)
(`docs/superpowers/decisions/2026-06-08-track2-first-scorecard-1b-vs-3b.md`
after that PR lands). Synthetic-only — real-mic far-field is GATED on founder
hardware (plan Task 9).

## Setup

- Branch `feat/v2-track2-stt-realsttfn` off main.
- Sidecar: locally-built `desktop/resources/sidecar` (MD5 `0ac1cdfd…`,
  same as PR #85 run; Mach-O arm64).
- Model: `~/.lisna-test-models/ggml-kotoba-whisper-v2.0-q5_0.bin`
  (Q5_0 GGML, 537,819,875 B, the kotoba JA-tuned whisper-v2.0 at q5).

  script — that script doubles as the reference).
- Noise bed: `desktop/tests/fixtures/audio/ja-bg-noise-30s.wav` (same
  format).
- Reference (135 chars):
  > 今日は良い天気ですね。これは日本語音声認識のテストです。コトバウィスパーというモデルを使って、サイドカープロセスで文字起こしを行います。三十秒ほど話し続けるので、しっかり認識できるか確認します。日本語の発音は明瞭で、機械翻訳と音声認識の両方に重要な役割を果たしています。
- Conditions: `clean`, `far-field-synth` (sparse-IR reverb + additive
  noise at SNR=5 dB, per `desktop/eval/stt/degrade.ts`).
- `far-field-real` deliberately NOT run — needs the founder mic
  recording (plan Task 9).

```bash
LISNA_TEST_STT_MODEL=~/.lisna-test-models/ggml-kotoba-whisper-v2.0-q5_0.bin \
pnpm --filter @lisna/desktop eval:stt --baseline v0-kotoba
```

Output saved to `desktop/eval/baselines/stt-v0-kotoba.json` (gitignored).

## Results

| condition | CER | WER | hyp Δ-summary |
|---|---|---|---|
| **clean** | **6.7 %** | 100 % | proper-noun rendering (コトバ→言葉), `三十秒`→`30秒`, kanji typos (`明瞭`→`明瞬`), one dropped suffix (`両方`→`両`) |
| **far-field-synth** (SNR 5 dB) | **17.8 %** | 100 % | all the clean errors + サイドカー→サイドパー + dropped 16-char clause (`機械翻訳と音声認識の両方に`) |

- Run wall time: 8,230 ms (one model load + 2 transcribe calls).
- **clean → far-field gap = 11.1 pp CER** — the headline number Phase 1
  routing depends on per
  [TRACK 2 spec §5](../specs/2026-06-08-v2-track2-quality-prioritization-design.md).
- **WER = 100 % is a metric artifact**, not a model failure. Japanese
  text has no inter-word spaces, so the WER tokenization in
  `desktop/eval/stt/metrics.ts` treats the whole utterance as one
  "word" and any character mismatch becomes 100 %. **For JA, only CER
  is meaningful.** (Spec mentioned this; recording here so the JSON
  baseline doesn't read as "everything failed" in future skims.)

## Reading

1. **Clean baseline 6.7 % CER on q5_0 is reasonable.** The misreads are
   bounded (one proper-noun reflex, two kanji slips, two minor token
   drops). Studio-quality JA capture is below the typical
   founder-perception threshold for "wrong" — 7 % CER reads as
   "essentially right" with stray polish-pass material.
2. **Far-field synth 17.8 % CER means capture matters.** A whole
   16-character clause vanished, plus a noun got morphed. At 17–18 %
   CER, downstream LLM structuring will start receiving sentences with
   missing referents — that's where note quality starts perceptually
   degrading.
3. **Phase 1 routing implication:** the 11.1 pp gap on synthetic alone
   is enough to argue that **capture is a non-trivial axis** (not "the
   model is broken; nothing else matters"). But synthetic is a
   **lower bound** on real CER (Lombard effect + actual room IR not
   modeled). The verdict isn't "capture dominates" until the
   `far-field-real` recording lands.

## Caveats

1. **Single fixture, single model, single SNR.** The plan calls out
   a model sweep (`kotoba q5_0`, `q8_0`, `whisper-large-v3`) and an
   SNR sweep for the actual Phase 0 exit; this is the wiring smoke,
   not the sweep itself.
2. **`far-field-synth` is sparse-IR + additive noise** — TRACK 2 spec
   §8 risk 2 explicitly flags this as a CER lower bound. Real recording
   needed for calibration (plan Task 9, founder hardware).
3. **JA WER stays 100 %.** Don't add a "WER regressed" check to CI for
   the JA path — it's noise. The CER row IS the signal.
4. **Cold Metal cache risk**: the kotoba load was the second use of
   this rebuilt sidecar today (3B Llama ran earlier — see [PR #85](https://github.com/May1350/Lisna/pull/85)).
   Cache state at run-start is "warm for some shaders, cold for STT-
   specific". A clean rebuild → kotoba-only re-run might shift the
   8.2 s wall time materially; don't lock latency conclusions on this
   single number (latency is downstream per the brainstorm anyway).

## Lessons learned (in addition to PR #85's)

- Re-using the existing `whisper-cpp-stt.test.ts` patterns (WAV header
  skip, `Int16Array → Float32Array` normalize, sidecar lifecycle) kept
  the new CLI thin — a 90-line script, all the heavy lifting in
  already-tested helpers (`runSttEval`, `degradeFarField`, `cer`).
- WER in `metrics.ts` is space-tokenized — for JA it's a 100 % flatline.
  Worth a one-line comment in the metrics file so future readers don't
  ship a "JA-WER-regression" alarm.
